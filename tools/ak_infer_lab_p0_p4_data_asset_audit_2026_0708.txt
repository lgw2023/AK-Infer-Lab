# 5. P0.5 vs Hardware Ceiling Methodology Comparison
| 差异项 | P0.5 microbench | hardware ceiling | 可能影响 | 是否有证据 |
| --- | --- | --- | --- | --- |
| source allocation | torch.empty CPU source, pageable only; .to(device) | torch.empty CPU source/destination; optional pin_memory() | pinned path exposed in new sweep | source code + CSV |
| destination allocation | implicit tensor returned by .to() or .cpu() | preallocated npu_dst / cpu_dst then copy_() | allocation behavior differs from copy measurement path | source code |
| copy API | source.to(device) and source.cpu() | npu_dst.copy_(cpu_src) / cpu_dst.copy_(npu_src) | not the same API path | source code |
| pinned | not parameterized | host_pinned 0/1 | new values can be much higher for large transfers | CSV 64 rows |
| non_blocking | only overlap path uses non_blocking=True | non_blocking 0/1 for every copy case | label does not imply async overlap due immediate synchronize | source code |
| synchronize | timed() calls torch.npu.synchronize() | timed() calls torch.npu.synchronize() after each fn() | both measure exposed synchronized latency | source code |
| warmup | once()+sync before timing | fn()+sync before timing each direction/config | similar high-level warmup, different API | source code |
| repetition count | 1 for >=256M else <=5 | 5 for all copy sizes in artifact | old large transfers less stable | source code + CSV |
| statistic | old stores averaged elapsed-derived bandwidth | new stores min and median but peak uses min/best | peak comparison is not median comparison | source code + CSV |
| size sweep | 4K..256M in old artifact | 4K..1G | new includes 1G and full matrix | CSV |
| device | AK_OBS_NPU_DEVICE default; 006 used server NPU env | npu:0 in run_context | different device/time may affect values | run_context |
| NUMA binding | not controlled | not controlled | NUMA source not isolated | no evidence |
| timing implementation | time.perf_counter + sync | time.perf_counter + sync | wall clock not NPU event timing | source code |
旧值保留价值: synthetic_microbench_measurement，代表早期 .to()/.cpu() pageable path 与 copy-overlap smoke；不适合作为当前 ceiling baseline。
新值替代边界: P0/P3 sweep 的 pinned large-copy min/best values 可作为 hardware_ceiling_baseline for current benchmark path；仍不是 PCIe/UB 理论值或模型 H2D/D2H exposed stall。

# 10. Matmul Observed Ceiling Analysis
FLOP formula: 2*M*N*K from source. Tensor allocation occurs outside timed fn. timed fn is left @ right, followed by torch.npu.synchronize(), measured with time.perf_counter wall clock. One warmup call precedes timed reps. 8192 runs 1 repetition by policy. Result is not explicitly consumed beyond launch/sync.
290.448949 TFLOPS can be called observed FP16 square-matmul ceiling for this benchmark path. It cannot be called Ascend 910B1 theoretical peak because no local theoretical spec evidence is present and external specs were not used.
