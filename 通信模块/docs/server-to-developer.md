# 服务器 → 开发人员结果交付模板

服务器完成任务后先在本地写 `result_summary.md` 并整理附件清单，在当前任务会话里等待用户为整个结果包选择 `email` 或 `upload-api`。确认前不得发送状态邮件正文、邮件测试、附件、upload-api 预检或文件。当前默认邮件收件人只使用 `yilili1023@gmail.com`，不再发送到 `gwlee1995@gmail.com`。`server-local` 只表示原始数据留存，不表示结果已交付。

选择 `email` 后，服务器使用 `../send_notify.py --confirmed-method email`，把 `result_summary.md` 作为邮件正文并在同一封邮件中附上已批准文件；正文和每个附件均不得超过 70KB。选择 `upload-api` 后，服务器不得补发状态邮件，而是把 `result_summary.md` 与已批准附件都作为文件上传。下面模板用于本地准备结果；未确认渠道时不得发送。

## 等待渠道确认（只在当前任务会话展示，不发送邮件）

```text
任务：
结果摘要文件：<服务器绝对路径、bytes、SHA-256、敏感性>
候选附件：<逐文件服务器绝对路径、bytes、SHA-256、敏感性>
可选对外交付：email / upload-api
建议方式及理由：
当前动作：所有结果保留 server-local，等待用户为上述完整范围选择渠道
```

## 任务完成

主题：`[AK服务器] 任务完成：<任务名>`

正文：

```text
任务：
主机：
时间：
结果摘要：
输出路径：
大文件留存路径：
后续建议：
```

用户确认只覆盖等待确认清单中列出的正文、附件范围和所选方式。若邮件或上传失败，只在当前任务会话给出脱敏失败摘要并重新等待选择；不要自动改名、重试、补发邮件或切换渠道。

## 通信配置变更或自检

主题：`[AK服务器] 通信配置更新：<范围>`

正文：

```text
主机：
时间：
变更范围：SMTP / proxychains / shell HTTP proxy / 收件人
是否涉及密钥轮换：是/否
脱敏配置摘要：
验证命令：
验证结果：
需要开发机同步的文档或代码：
```

注意：不要在邮件正文或附件中发送 `.env`、SMTP 授权码、代理账号、代理密码、服务器账号、私钥、Cookie 或其他敏感信息。需要共享密钥时，只说明“服务器本地 .env 已更新”或“proxychains 本机配置已更新”。

附件注意：只有用户为完整结果包明确选择 `email` 后才发送邮件，且正文与每个附件均不得超过 70KB。任何超过 70KB 的日志、zip、profiler、trace、数据集、模型输出或实验目录默认留在昇腾服务器本地；如用户选择 `upload-api`，按 `file-transfer-policy.md` 上传 `result_summary.md` 与全部已批准附件，不额外发送邮件。
