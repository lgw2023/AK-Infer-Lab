# 服务器 → 开发人员邮件模板

服务器使用 `../send_notify.py` 向开发人员发送邮件。当前默认收件人只使用 `yilili1023@gmail.com`，不再发送到 `gwlee1995@gmail.com`。昇腾服务器每次邮件正文和每个附件都按不超过 70KB 处理；大规模实验数据、raw profiler、完整日志、模型输出和实验目录默认留在服务器本地，只在邮件中回传摘要、小清单、少量样例和服务器路径。若需要回传文件，`email`、`upload-api`、`server-local` 是三个候选方式，必须先让用户针对具体文件选择；状态正文可正常发送，但未确认前不得附文件或上传。下面模板可作为主题和正文参考。

## 任务完成

主题：`[AK服务器] 任务完成：<任务名>`

正文：

```text
任务：
主机：
时间：
结果摘要：
输出路径：
大文件留存路径：
后续建议：
```

## 文件传输方式待用户确认

主题：`[AK服务器] 待确认文件传输方式：<任务名>`

正文：

```text
任务：
主机：
时间：
待传文件绝对路径或有界清单：
逐文件大小（bytes + 人类可读）：
敏感性：普通 / 可能敏感 / 敏感
可选方式：email / upload-api / server-local
建议方式及理由：
超过 100MB 或其他风险：
当前动作：等待用户明确选择，不附文件、不上传、不自动切换通道
```

用户确认只覆盖邮件中列出的文件范围和所选方式。若上传返回 `401`、`409`、`413`、代理/重定向、超时、服务异常或 SHA-256 不一致，只发小体积失败摘要并重新等待选择；不要自动改名、重试或改用邮件。

需要开发机补充的信息：
```

## 资源告警

正文：

主机：

资源类型：CPU/NPU/内存/磁盘/网络
当前状态：
影响范围：
建议处理：
```

## 通信配置变更或自检

主题：`[AK服务器] 通信配置更新：<范围>`

正文：

```text
主机：
时间：
变更范围：SMTP / proxychains / shell HTTP proxy / 收件人
是否涉及密钥轮换：是/否
脱敏配置摘要：
验证命令：
验证结果：
需要开发机同步的文档或代码：
```

注意：不要在邮件正文或附件中发送 `.env`、SMTP 授权码、代理账号、代理密码、服务器账号、私钥、Cookie 或其他敏感信息。需要共享密钥时，只说明“服务器本地 .env 已更新”或“proxychains 本机配置已更新”。

附件注意：只有用户明确选择 `email` 后才发送附件，并且只发送 70KB 以内的小文件。任何超过 70KB 的日志、zip、profiler、trace、数据集、模型输出或实验目录默认留在昇腾服务器本地，邮件正文说明服务器路径、文件大小和建议的后续分析任务；如用户选择 `upload-api`，再按 `file-transfer-policy.md` 执行。
