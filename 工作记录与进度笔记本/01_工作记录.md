# 01 工作记录

## 总目标

构建一套面向模型推理的硬件负载观测与瓶颈归因工作流：基于标准推理框架和静态提示词 workload，分析模型推理在 NPU、HBM、CPU、DRAM、H2D/D2H、PCIe/UB、SSD、KV/prefix 和 MoE expert 等环节上的压力，并补齐状态对象级 trace、运行时队列/调度 trace、copy 与 compute overlap trace，判断瓶颈来自哪里。

## 范围边界

### 范围内

- 标准模型推理链路：prefill、decode、attention、MLP/MoE、KV cache、sampling。
- 静态提示词 workload：用长上下文、多轮对话、代码修改请求、工具输出文本等提示词近似 coding-agent 类请求。
- 硬件链路观测：AI Core/AIV、HBM 容量和带宽、CPU 调度和指令、DRAM 带宽、H2D/D2H、PCIe/UB、SSD I/O、copy stream、sync barrier 和 overlap。
- 状态对象观测：KV、Prefix、Expert、Weight、Activation、Workspace 的位置、生命周期、迁移、命中、恢复、重算和驱逐。
- 运行时观测：请求队列、batch 形成、prefill/decode 阶段、cache manager、policy decision、stall reason。
- MoE 行为观测：router top-k、expert 热度、reuse distance、expert tier hit/miss、prefetch lead time 和 miss penalty。
- trace-driven simulator：用真实 trace 和 microbench 数据做瓶颈归因与 what-if 分析。

### 范围外

- 真实 coding agent 运行。
- 工具调用系统、终端执行、浏览器自动化、文件读写代理。
- 真实仓库扫描、代码补丁生成质量评估。
- agent 调度器、多 agent 协作、任务规划质量评估。
- 一开始就挑战 GLM5.2 满血低时延、NPU-SSD 直通或 CPU 替代 NPU 主算。
