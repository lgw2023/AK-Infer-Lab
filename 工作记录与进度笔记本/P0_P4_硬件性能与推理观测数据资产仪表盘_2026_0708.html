5. Qwen3.5-4B / vLLM 推理观测链路

推理观测结论Qwen3.5-4B 这轮最有价值的是把请求、prefill/decode、msprof task/op 读数连起来，并把输出长度固定住。它证明 raw readout 链路可用；还没有证明 prefix cache 让用户可感知延迟变好。
模型视角
当前证据只确认 Qwen3.5-4B 的服务路径和 profiler 读数链路；未补充层数、hidden size、head 数等未归档架构参数。
推理过程视角
按请求进入、prefill 建立上下文、decode 逐步生成、msprof 采集 NPU task/op 的顺序理解，不把 raw counter 等同于延迟收益。

6. Claim Readiness 与 30 个关键问题

证据分级这 30 个问题是把“能讲什么”和“还不能讲什么”拆开：硬件 inventory 与 microbench ceiling 已经可讲，瓶颈、利用率、TTFT/TPOT 收益和 DeepSeek 优先级还不能讲。这个分级能防止下一阶段把漂亮数字误读成优化结论。
YES能否把 H2D 作为主机到 NPU 的传输上限基线？可以。当前最佳观测值 24.313915 GB/s，来自 1G pinned 同步路径。
YES能否把 D2H 作为 NPU 回传主机的传输上限基线？可以。当前最佳观测值 26.480714 GB/s，来自 1G pinned 同步路径。
YES能否说明 copy 吞吐随 payload size 明显变化？可以。64 行 sweep 覆盖 4K 到 1G、H2D/D2H、pinned/non_blocking 组合。
YES能否说 pinned host memory 对 H2D 有观测收益？可以带条件说。当前矩阵 median speedup 为 1.694x。
YES能否说 pinned host memory 对 D2H 有观测收益？可以带条件说。当前矩阵 median speedup 为 1.637x，但个别大块组合为负。
PARTIAL能否用 non_blocking 结果说明异步传输收益？只能部分说明。每次 timed call 后立即 synchronize，因此不是 host-side overlap 证据。
NO能否说 host 端异步提交能力已经测清楚？不能。当前没有独立测 host issue latency 或队列深度行为。
PARTIAL能否说明 copy-copy overlap 是否存在？只能作为候选线索。当前主要依赖旧 smoke，不足以作为正式结论。
NO能否说明 copy 能被 compute 隐藏？不能。当前没有 copy-compute overlap event microbench。
NO能否判断 CPU-NPU 最佳 NUMA 亲和关系？不能。当前没有跨 NUMA 绑定矩阵。
NO能否把 NumPy DRAM 读数当成服务器 DRAM 硬件峰值？不能。它只是 NumPy 路径 ceiling，不是内存子系统理论峰值。
YES能否把 NumPy memory path 当作软件路径基线？可以。当前 read path 最佳 5.332 GB/s，可作为该 Python/NumPy 路径参考。
YES能否把 FP16 square matmul 作为 NPU compute observed ceiling？可以。8192³ 单次观测为 290.449 TFLOPS。
NO能否判断 Qwen 真实 GEMM 效率？不能。缺少 Qwen runtime 中 MatMul shape/FLOPs denominator。
NO能否给出 AI Core 利用率百分比？不能。当前只有 raw counters，没有单位和有效 denominator。
PARTIAL能否比较 AIC、AIV、MTE raw 变化方向？可以部分比较 raw sums，但不能转成瓶颈或利用率。
YES能否列出 MAC/MTE/vector 相关 raw delta？可以。附录保留 21 行 AI Core/AIV/MTE metric delta。
NO能否说明 decode 阶段 memory-bound？不能。没有 HBM bytes/utilization denominator，也没有 token-level attribution。
NO能否说明 HBM 带宽是否成为瓶颈？不能。当前未测 HBM bandwidth/utilization。
PARTIAL能否说明 KV cache 已被观测到？只能说明存在相关统计和 prefix 分组，不能说明压力或命中率。
NO能否判断 KV cache pressure？不能。缺 KV 容量、页表、eviction、bytes 等压力指标。
PARTIAL能否判断 scheduler/queue pressure？只能部分判断。当前 request stats 有线索，但不是完整排队模型。
PARTIAL能否解释 request wait time？只能按当前 raw stats 陈述；zero wait 不等于无调度瓶颈。
YES能否把 request window 与 device task 聚合关联起来？可以。已有 aggregate path 和 request-device 对齐读数。
YES能否把 request 与 op type 聚合关联起来？可以。已有 top-op aggregate，但只是 raw delta 候选列表。
NO能否定位慢 token 对应的具体 op？不能。当前没有 token-level timeline 到 op 的映射。
NO能否证明 prefix cache 改善 TTFT？不能。P1.28 是 raw counter readout，不是 TTFT benchmark。
NO能否证明 prefix cache 改善 TPOT？不能。没有稳定 TPOT 对照和服务生命周期闭环。
NO能否给出 top-3 stall reason？不能。缺单位映射、shape/FLOPs/bytes denominator 和时序归因。
NO能否据此决定 DeepSeek P8/P9 优化优先级？不能。Qwen 小模型 raw delta 不能迁移为 DeepSeek MoE/EP/MTP 结论。
