## 7. P8：分层工程原型

### 7.1 总体口径

P8 不是“一步到位写完整框架”，而是四层组合：

1. **运行底座**：vLLM-Ascend 主路；MindIE availability-gated 对照路。
2. **可复用能力**：Prefix Cache、KV Cache CPU Offload、UCM/External KV Cache、EPLB/static expert map。
3. **自研控制面**：统一 `StateObject`、`StateEvent`、`PlacementDecision`，但不持有 tensor payload。
4. **实验与仿真**：KV real path、expert trace/static placement、Expert Tier V0、条件式 warm prefetch、P9 trace bundle。

### 7.2 StateObject 范围

统一对象：

```text
KV block
Prefix block
Expert weight
Weight shard
Session
```

Trace 是事件，不是对象；whole-device HBM、tensor footprint 或 server stats proxy 不能冒充 object bytes。

### 7.3 P8 垂直切片

| 阶段 | 目标 | 退出门 |
| --- | --- | --- |
| P8.0 | capability matrix + baseline contract | 选中路径不再是 `documented_unverified` |
| P8.1 | observe-only 统一 trace | `trace_validation_errors=0`，关键 join 可用 |
| P8.2 | Prefix/KV CPU Offload/UCM DRAM-first | real path、hit/miss、bytes/latency 或明确 unknown |
| P8.3 | expert hotness + expert map + static placement | workload 固定、expert identity 和分布稳定 |
| P8.4 | Expert Tier V0 simulation | holdout replay、miss penalty、误差与 provenance 完整 |
| P8.5 | DRAM→HBM warm prefetch V1 | 仅在 lead time、load cost、wrong-prefetch、HBM budget 门通过后启动 |
| P8.6 | P9 trace bundle | P9 可机器读取，不需从 Markdown 抄数 |

### 7.4 KV/Prefix 顺序

```text
P6 Prefix baseline
  -> KV Cache CPU Offload
  -> UCM/External KV Cache DRAM-first
  -> SSD/NFS/3FS cold persistence
  -> MindIE comparison when available
```

SSD 不进入逐 token decode 热路径。路径跑通但性能为负仍是有效结果。

### 7.5 MoE 顺序

```text
aggregated expert hotness
  -> EPLB recording/static map
  -> request/session/router trace only when needed
  -> Expert Tier V0 simulation
  -> gated warm prefetch V1
```

EPLB 只证明 load balance/placement，不等于 expert offload。没有 expert bytes、load latency、miss penalty 时不声称分层收益。
