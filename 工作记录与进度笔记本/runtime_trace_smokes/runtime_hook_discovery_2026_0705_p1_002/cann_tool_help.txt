Usage:
      msprof [--options]

This is subcommand for operator optimization situation:
      op                                Use binary msopprof to operator optimization (msprof op ...)

Options:
      --output                          <Optional> Specify the directory that is used for storing data results.
      --application                     <Optional> Specify application path, considering the risk of privilege escalation, please pay attention to
						   the group of the application and confirm whether it is the same as the user currently.
						   [Note] This option will be discarded in later versions.
						   you can try to use: msprof [msprof arguments] <app> [app arguments]
      --ascendcl                        <Optional> Show acl profiling data, the default value is on.
      --ge-api                          <Optional> Specify if report GE event, the default value is off. The possible parameters are 'l0', 'l1' or 'off'.
      --runtime-api                     <Optional> Show runtime api profiling data, the default value is off.
      --task-time                       <Optional> Show task profiling data, the default value is on. The possible parameters are 'l0', 'l1', 'l2', 'on' or 'off'.
      --task-memory                     <Optional> Show the memory usage of the operator, the default value is off. The possible parameters are 'on' or 'off'.
      --ai-core                         <Optional> Turn on / off the ai core profiling, the default value is on when collecting app Profiling.
      --aic-mode                        <Optional> Set the aic profiling mode to task-based or sample-based.
						   In task-based mode, profiling data will be collected by tasks.
						   In sample-based mode, profiling data will be collected in a specific interval.
						   The default value is task-based in AI task mode, sample-based in system mode.
      --aic-freq                        <Optional> The aic sampling frequency in hertz, the default value is 100 Hz, the range is 1 to 100 Hz.
      --environment                     <Optional> User app custom environment variable configuration.
      --sys-period                      <Optional> Set total sampling period of system profiling in seconds.
      --sys-devices                     <Optional> Specify the profiling scope by device ID when collect sys profiling.The value is all or ID list (split with ',').
      --hccl                            <Optional> Show hccl profiling data, the default value is off. [Note] This option will be discarded in later versions.
      --msproftx                        <Optional> Show msproftx and mstx data, the default value is off.
      --mstx-domain-include             <Optional> Choose to only include mstx events from a comma separated list of domains;
						   `default` filters the mstx default domain;
						   The switch is only applicable when parameter msproftx is set to on;
						   The switch cannot be set with mstx-domain-exclude at the same time.
      --mstx-domain-exclude             <Optional> Choose to exclude mstx events from a comma separated list of domains;
						   `default` excludes the mstx default domain;
						   The switch is only applicable when parameter msproftx is set to on;
						   The switch cannot be set with mstx-domain-include at the same time.
      --storage-limit                   <Optional> Specify the output directory volume. range 200MB ~ 4294967295MB.
      --model-execution                 <Optional> Show ge model execution profiling data, the default value is off. [Note] This option will be discarded in later versions.
      --aic-metrics                     <Optional> The aic metrics groups, include ArithmeticUtilization, PipeUtilization, Memory, MemoryL0, ResourceConflictRatio, MemoryUB, L2Cache, MemoryAccess.
						   the default value is PipeUtilization.
      --python-path                     <Optional> Specify the python interpreter path that is used for analysis, please ensure the python version is 3.7.5 or later.
      --parse                           <Optional> Switch for using msprof to parse collecting data, the default value is off.
      --query                           <Optional> Switch for using msprof to query collecting data, the default value is off.
      --export                          <Optional> Switch for using msprof to export collecting data, the default value is off.
      --clear                           <Optional> Switch for using msprof to analyze or export data in clear mode, the default value is off.
      --analyze                         <Optional> Switch for using msprof to analyze collecting data, the default value is off.
      --rule                            <Optional> Switch specified rule for using msprof to analyze collecting data, include communication, communication_matrix.
						   the default value is communication,communication_matrix.
      --iteration-id                    <Optional> The export iteration id, only used when argument export is on, the default value is 1.
      --model-id                        <Optional> The export model id, only used when argument export is on, msprof will export minium accessible model by default.
      --summary-format                  <Optional> The export summary file format, only used when argument export is on, include csv, json, the default value is csv.
      --type                            <Optional> The export type, only used when argument `export` is on
						   or when collecting data, include text, db.
						   the default value is text(which will also export the database).
      --reports                         <Optional> Specify the path that is used for controlling the export scope of collecting results
      --aicpu                           <Optional> Show aicpu profiling data, the default value is off.
      --sys-hardware-mem                <Optional> LLC, DDR, HBM acquisition switch, optional on / off, the default value is off.
      --sys-hardware-mem-freq           <Optional> LLC, DDR, HBM acquisition frequency, range 1 ~ 100, the default value is 50, unit Hz.
      --llc-profiling                   <Optional> The llc profiling groups, include read, write. the default value is read.
      --sys-cpu-profiling               <Optional> The CPU acquisition switch, optional on / off,the default value is off.
      --sys-cpu-freq                    <Optional> The cpu sampling frequency in hertz. the default value is 50 Hz, the range is 1 to 50 Hz.
      --sys-profiling                   <Optional> The System CPU usage and system memory acquisition switch,the default value is off.
      --sys-sampling-freq               <Optional> The sys sampling frequency in hertz. the default value is 10 Hz, the range is 1 to 10 Hz.
      --sys-pid-profiling               <Optional> The CPU usage of the process and the memory acquisition switch of the process,the default value is off.
      --sys-pid-sampling-freq           <Optional> The pid sampling frequency in hertz. the default value is 10 Hz, the range is 1 to 10 Hz.
      --sys-io-profiling                <Optional> NIC, ROCE acquisition switch, the default value is off.
      --sys-io-sampling-freq            <Optional> NIC, ROCE acquisition frequency, range 1 ~ 100, the default value is 100, unit Hz.
      --instr-profiling                 <Optional> Show instr profiling data, the default value is off.
      --instr-profiling-freq            <Optional> The instr profiling sampling period in clock-cycle, the default value is 1000 cycle, the range is 300 to 30000 cycle.
      --sys-interconnection-profiling   <Optional> PCIE, HCCS acquisition switch, the default value is off.
      --sys-interconnection-freq        <Optional> PCIE, HCCS acquisition frequency, range 1 ~ 50, the default value is 50, unit Hz.
      --dvpp-profiling                  <Optional> DVPP acquisition switch, the default value is off.
      --dvpp-freq                       <Optional> DVPP acquisition frequency, range 1 ~ 100, the default value is 50, unit Hz.
      --l2                              <Optional> L2 Cache and SMMU acquisition switch. The default value is off.
      --host-sys                        <Optional> The host-sys data type, include cpu, mem, disk, network, osrt
      --host-sys-pid                    <Optional> Set the PID of the app process for which you want to collect performance data.
      --host-sys-usage                  <Optional> The host-sys-usage data type, include cpu, mem.(full-platform)
      --host-sys-usage-freq             <Optional> The sampling frequency in hertz. the default value is 50 Hz the range is 1 to 50 Hz.
      --dynamic                         <Optional> Dynamic profiling switch, the default value is off.
      --pid                             <Optional> Dynamic profiling pids of the target processes
      --delay                           <Optional> Collect start delay time in seconds, range 1 ~ 4294967295s.
      --duration                        <Optional> Collection duration in seconds, range 1 ~ 4294967295s.
      --help                            <Optional> help message.
msnpureport=/usr/bin/msnpureport
Recommend use new command, use 'msnpureport help' to get more information.
Usage:	msnpureport
  or:	msnpureport[OPTIONS]
Example:
	msnpureport                     Export all log and file.
	msnpureport -a                  Export all log and file, device event information in the bbox
	msnpureport -f                  Export all log and file, device event information in the bbox, historical maintenance and measurement information in the bbox
	msnpureport -t 1                Export device log, include slog, message and system_info
	msnpureport -g info             The device (which device id is 0) global log level options to info
	msnpureport -m FMK:debug -d 1   The device (which device id is 1) FMK module log level options to debug
	msnpureport -r                  Get log level
Options:
    --docker             Docker flag, msnpureport in docker must be explicitly used with this flag
    -g, --global         Set device global log level: error, info, warning, debug, null
    -m, --module         Set device module log level: error, info, warning, debug, null
    -e, --event          Set device event log level: enable, disable
    -t, --type           Export type: 0(all log and file, default is 0), 1(slog, message and system_info), 2(bbox, current register), 3(stackcore), 4(vmcore), 5(module log)
    -d, --device         Device ID, default is 0, can be used only with -g or -m or -e or -r
    -r, --request        Get device log level
    -a, --all            Export all log and file, device event information in the bbox
    -f, --force          Export all log and file, device event information in the bbox, historical maintenance and measurement information in the bbox
    -h, --help           Help information
